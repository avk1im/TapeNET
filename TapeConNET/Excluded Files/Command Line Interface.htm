tapecon Command Line

The tapecon command line consists of one or several flags each optionally followed by options. General command line format:

tapecon ‑flag1 [options..] [-flag2 [options..] ..]

There are task flags and mode flags. The task flags direct tapecon to fulfill certain tasks, for example ‑backup to backup files or ‑restore to restore files. The mode flags instruct tapecon how to fulfill the following tasks, for example ‑subdirectories mode flag instructs to recurse subdirectories when performing the ‑backup task.

The same mode flag can apply to several tasks. For example, ‑subdirectories mode flag also instructs tapecon to restore files with subdirectories when performing the ‑restore task.

 

Command Line Syntax

General command line format:

tapecon ‑flag1 [options..] [-flag2 [options..] ..]

General flag format: ‑flag [options..]

Alternative formats to specify a single option (all equivalent): ‑flag <option>, ‑flag:<option>, or ‑flag=<option>

Flags and options are NOT case‑sensitive. Therefore, e.g., ‑flag:on, ‑flag:ON, ‑Flag:on, and ‑FLAG:On are all equal. The options that are file and directory (folder) names preserve the case for file operations.

The flags and options are processed sequentially from left to right. Therefore, make sure to specify the mode flags related to a certain task before the flag for the task. For example, to execute a backup task recursing through subdirectories, specify:

tapecon ‑subdirectories ‑backup C:\ToBackup

To perform several tasks in one invocation of tapecon, specify them sequentially. For example, to perform a validation pass after the backup, specify:

tapecon ‑subdirectories ‑backup C:\ToBackup ‑validate

There are multiple equivalent ways to specify on/off flags:

       Switching a mode on (all equivalent): ‑flag, ‑flag on, ‑flag:on, ‑flag=on, ‑flag+

       Switching a mode off (all equivalent): ‑flag off, ‑flag=off, ‑flag:off, ‑flag‑

Task Flags

Drive Selection Task:

  • Use ‑drive [drive_number] or ‑d [drive_number] task flag to specify the tape drive to open and use for the subsequent tasks.
  • Alternative formats: ‑drive:<drive_number>, ‑drive=<drive_number>
  • Specify the drive number as an integer from 0 on. If no number is specified, 0 is assumed.
  • Include this flag before all other flags handling the drive.
  • If the flag has not been specified before the first task flag, tapecon will attempt to open and use tape drive 0.

Format Task:

  • To format the tape in the drive, use ‑format task flag.
  • WARNING: Formatting a tape will irreversibly destroy all data on the tape!
  • By default, on the drives that support multiple partitions, the tape will be formatted with two partitions: for the content and for the table of content (TOC), as this provides the optimal performance and reliability. To enforce formatting with a single partition and so placing the TOC in the same partition as the content, specify single option. This might help decrease tape wear, at the expense of somewhat slower access to TOC.

Backup Task:

  • To initiate a backup, use the ‑backup [files..] task flag.
  • Following the flag, specify the source files and/or directories to back up. You can use wildcards for directories and files.
  • Optionally, before the backup flag include a ‑subdirectories, ‑subfolders, ‑subdir, or ‑s on/off flag to recurse through subdirectories.
  • Choose whether to append the new backup set to existing ones or replace all previous backups on the tape using the on/off flag ‑append or ‑a.
  • Optionally, use data integrity verification flag ‑crc or ‑c to specify a hash (CRC) verification algorithm for the new backup set (s. below).
  • Remember to provide all backup mode specification flags before the backup task flag itself.
  • Data Integrity Verification (CRC) mode flag:
  • Specify the hash (CRC) verification algorithm using ‑crc <algorithm> or ‑c <algorithm>
    • Example: ‑c crc64
    • Use ‑hash none or ‑hash off to skip CRC verification.
    • For possible hash algorithm values, s. Data Integrity Verification (CRC) Mode

Restore Task:

  • To restore data, use the ‑restore [set_number] [files..] flag.
  • Alternative formats: ‑restore:<set_number>, ‑r=<set_number>
  • By default, the most recent backup set gets restored. Optionally specify the number of the backup set to restore (if multiple sets exist on the tape):
    • Positive number from 1 to <number_of_sets>: from the oldest (1) to the most recent (<number_of_sets>) set
    • 0 or no number specified: the most recent set
    • Negative number from ‑1 to <-(number_of_sets ‑ 1)>: from the second most recent (-1) to the oldest (<-(number_of_sets ‑ 1)>)
  • Optionally include one or more file names, with our without wildcards, to restore only the matching files
  • Choose whether to write files to the original location or a specified alternative location using ‑target <path> or ‑t <path>.
    • If the directory name is missing, the target directory setting is cleared: files will be restored to the original path
  • Optionally, set a ‑subdirectories, ‑subfolders, or ‑s flag to restore the directory structure for the files.
    • If an alternative target directory is specified, the directory structure will be restored under it
  • Specify how to handle existing files using the ‑existing or ‑e flag with values:
    • overwrite or o: Overwrite existing files during restore.
    • skip or s: Skip existing files.
    • keepboth, keep, both, k, or none (default value): Keep both copies (rename restored files).

Verification Task:

  • Use ‑verify [set_number] [files..] or ‑y flag to perform a verification pass.
  • For the explanation of the set_number and files parameters, s. description of Restore Task
  • Verification compares the backed‑up files on tape with the original ones. To perform the validation of the backed‑up files without comparing them with the original ones, use Validation Task

Validation Task:

  • Use ‑validate [set_number] [files..] or ‑v flag to perform a validation pass.
  • For the explanation of the set_number and files parameters, s. description of Restore Task
  • The backed‑up files are only validated using the hash (CRC) algorithm no comparison with the original files is performed. To perform verification comparing the backed‑up files on tape with the original ones, use Verification Task.

List Task:

  • Use ‑list [set_number] [files..] or ‑l flag to list the content of a particular backup set or the whole media.
  • For the explanation of the set_number and files parameters, s. description of Restore Task
  • Optionally, set a ‑subdirectories, ‑subfolders, or ‑s option flag to list the full file names including the directory structure. Otherwise, only file names will be listed.

Additional Tasks:

  • For other tasks beyond backup and restore:
    • ‑help or ‑?: Display usage instructions.

 

Command Line Examples

Backup with recursion, appending the new backup set to existing backups, with CRC verification using crc64, then validate the new backup set using CRC values:

tapecon ‑d 0 ‑s ‑a ‑c crc64 ‑b "C:\Data\*" ‑v crc

Restore the third oldest backup set to a specified directory, overwriting existing files and restoring the directory structure:

tapecon ‑d 0 ‑t "D:\RestoredFiles" ‑s on ‑e overwrite ‑r 3

Validate the 2nd most recent backup set using only saved CRC values:

tapecon ‑d 0 ‑validate crc ‑1
